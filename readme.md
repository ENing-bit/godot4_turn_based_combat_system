##  玩法调优方案：让战斗“活”起来

我不太会godot引擎，所以不好直接调整，我只好作为一个策划的角度来看这款游戏：

原版战斗系统结构清晰，但策略 depth 和节奏感有提升空间。我提出以下3组改动方案：

### 1. 行动点（AP）系统
**目标：** 用资源管理替代简单的一回合一动，增加每回合的决策 weight。
**方案：** 每回合给予角色3点行动点，攻击、技能、防御消耗不同点数。
**预期效果：** 玩家需权衡“爆发”与“稳健”，策略选择更丰富。

### 2. 连击（Combo）机制
**目标：** 鼓励进攻，为战斗创造节奏起伏。
**方案：** 连续攻击叠加伤害加成，中断（防御/治疗）则重置。
**预期效果：** 制造“高风险高回报”的决策点，让战斗过程更具情绪张力。

### 3. 破防/韧性机制
**目标：** 提升控制技能的战略价值，丰富战斗维度。
**方案：** 控制技能削减敌人韧性条，归零后敌人进入易伤状态。
**预期效果：** 将控制技能从“可选”变为“战术核心”，创造 combo 窗口期。

**设计原则：** 这三组改动均遵循“为玩家创造有意义的决策”这一核心目标，从资源管理、战斗节奏、战术 depth 三个维度提升战斗系统的可玩性。

###心得： Fork 这个项目是因为我想找一个看得懂的战斗系统来练手。
原版结构清晰，但玩了几局之后有个明显感受：能玩，但不太想玩第二局。 问题出在战斗的决策空间上——“一回合一动”让每回合的选择过于独立，缺少策略的累积感。玩家的行动之间没有联动，就很难产生“我这一套打得漂亮”的爽感。
所以我提出了三组改动方向：行动点系统让玩家自己分配资源，从“选什么”变成“怎么组合”；连击系统把进攻变成需要维护的资源，制造节奏起伏；破防/韧性系统让控制技能从“选项”变成“战术组件”。
